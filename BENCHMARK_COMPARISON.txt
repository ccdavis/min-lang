WHY THE DIFFERENCE?

Startup time components (estimated):
  • Lexer initialization:        ~5ms
  • Parser setup:                 ~10ms
  • Compiler warmup:              ~15ms
  • VM initialization:            ~5ms
  • First function call overhead: ~10ms
  
Total startup overhead:           ~45ms

Impact on benchmarks:
  • Quick (457ms total):  45ms = 9.8% overhead
  • Heavy (27,680ms total): 45ms = 0.16% overhead

MinLang appears faster in quick tests because:
  ✗ Startup cost is proportionally larger
  ✗ Parser/compiler haven't warmed up caches
  ✗ VM hasn't optimized instruction dispatch patterns

Python appears slower in quick tests because:
  ✗ CPython has higher startup cost (~30-50ms)
  ✗ Import machinery, module loading
  ✗ More complex initialization

Heavy benchmark reveals TRUE performance:
  ✓ Startup time becomes negligible
  ✓ Pure computational performance measured
  ✓ Cache effects stabilized
  ✓ Instruction dispatch patterns optimized
